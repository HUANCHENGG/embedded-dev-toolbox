关于数据校验

什么是数据校验？

数据校验（Data Checksum / Error Detection）是在数据传输或存储过程中，通过对原始数据进行特定数学运算生成一个校验值，接收方用同样的算法重新计算并比对，以检测数据是否在传输中发生错误或被篡改。

BCC / XOR 异或校验

BCC（Block Check Character）是最简单的校验方式，将所有数据字节逐一进行异或（XOR）运算，得到一个单字节校验值。计算速度极快，适用于简单的串口通信协议。

  • 优点：实现简单、计算快、占用资源极少
  • 缺点：检错能力弱，无法检测偶数个比特同时翻转的错误
  • 应用：RS-485 通信、简单嵌入式协议、IC 卡通信（ISO 14443）

CRC 循环冗余校验

CRC（Cyclic Redundancy Check）基于多项式除法原理，将数据视为一个大的二进制数，除以一个预定义的生成多项式，余数即为 CRC 校验值。CRC 具有很强的检错能力，能检测所有单比特错误、双比特错误和奇数个比特错误。

CRC 参数说明

  • Poly（多项式）：生成多项式的简写形式，省略最高位 1
  • Init（初始值）：CRC 寄存器的初始值
  • RefIn（输入反转）：是否对每个输入字节进行位反转
  • RefOut（输出反转）：是否对最终 CRC 值进行位反转
  • XorOut（结果异或）：最终 CRC 值与此值异或后输出

累加和 Checksum

累加和是将所有数据字节相加，取结果的低位作为校验值。变体包括直接截取、取反加一（补码）、进位回卷等。

  • 直接截取：将累加结果截取到指定位宽（8/16/32 位）
  • 取反+1：对截取结果取补码，常用于校验和验证（所有数据加上校验值结果为 0）
  • 进位回卷：溢出的高位加回低位，TCP/UDP/IP 协议使用此算法

关于编码转换

数制（进制）基础

数制是用一组固定的符号和规则来表示数值的方法。计算机内部使用二进制，但人类更习惯十进制，因此不同进制之间的转换是编程中的基本操作。

进制基数字符集前缀标识典型用途
二进制20-10b硬件寄存器、位操作
八进制80-70o 或 0Unix 文件权限（chmod 755）
十进制100-9无日常计算、用户界面
十六进制160-9, A-F0x内存地址、颜色值、字节数据

ASCII 编码

ASCII（American Standard Code for Information Interchange）是最基础的字符编码标准，使用 7 位二进制数（0-127）表示 128 个字符，包括 33 个控制字符和 95 个可打印字符。

  • 0x00-0x1F：控制字符（如 0x0A 换行、0x0D 回车）
  • 0x20-0x7E：可打印字符（空格、字母、数字、标点）
  • 0x30-0x39：数字 0-9
  • 0x41-0x5A：大写字母 A-Z
  • 0x61-0x7A：小写字母 a-z

Base64 编码

Base64 是一种将二进制数据编码为 ASCII 字符串的方法。它将每 3 个字节（24 位）分为 4 组 6 位数据，每组映射到一个可打印字符。编码后数据量增大约 33%。

  • 标准字符集：A-Z、a-z、0-9、+、/，填充字符 =
  • URL 安全变体：将 + 替换为 -，/ 替换为 _，省略填充 =
  • 应用场景：电子邮件附件（MIME）、Data URI、JWT Token、API 传输二进制数据

UTF-8 编码

UTF-8 是 Unicode 的一种变长编码实现，使用 1-4 个字节表示一个字符，兼容 ASCII。它是互联网上最广泛使用的字符编码。

关于字符串与编码长度

字符与字节的区别

在计算机中，"字符"和"字节"是两个不同的概念。一个字符是人类可读的最小文本单位（如一个汉字、一个字母），而字节是计算机存储的最小单位。不同编码下，一个字符可能占用不同数量的字节。

各编码下的字节占用

字符类型ASCIIUTF-8UTF-16GBK
英文字母/数字1 字节1 字节2 字节1 字节
中文汉字不支持3 字节2 字节2 字节
Emoji 表情不支持4 字节4 字节（代理对）不支持
日文假名不支持3 字节2 字节不支持

字符长度 vs 字节长度

  • 字符长度：文本中字符的个数，与编码无关。JavaScript 中 "你好".length 返回 2
  • 字节长度：文本在特定编码下占用的字节数。"你好" 在 UTF-8 下为 6 字节，GBK 下为 4 字节
  • 显示宽度：在等宽字体终端中的显示列数。全角字符（中文、全角标点）占 2 列，半角字符占 1 列

JavaScript 中的字符串长度

JavaScript 内部使用 UTF-16 编码。String.length 返回的是 UTF-16 码元（code unit）的数量，而非字符数。对于 BMP 之外的字符（如 Emoji），一个字符占用 2 个码元（代理对），导致 length 返回 2。

字符数 [..."你好🌍"].length → 3
码元数 "你好🌍".length → 4
UTF-8字节 new TextEncoder().encode("你好🌍").length → 10

常见应用场景

  • 数据库字段：VARCHAR(n) 中 n 通常指字符数，但实际存储受编码影响
  • 网络传输：HTTP Content-Length 头指的是字节数，非字符数
  • 短信/推文：字符数限制通常按 Unicode 字符计算
  • 终端显示：对齐表格时需考虑全角/半角的显示宽度差异
  • 嵌入式开发：内存受限设备需精确计算字符串的字节占用

关于哈希与加密

哈希（散列）与加密的区别

哈希和加密是两种不同的密码学操作。哈希是单向的，无法从哈希值还原原始数据；加密是双向的，使用密钥可以将密文还原为明文。

特性哈希（Hash）对称加密（如 AES）
方向单向，不可逆双向，可加密可解密
输出长度固定长度与输入长度相关
是否需要密钥不需要需要
用途完整性校验、密码存储、数字签名数据保密传输与存储

MD5 消息摘要算法

MD5（Message-Digest Algorithm 5）由 Ronald Rivest 于 1991 年设计，输出 128 位（16 字节）哈希值，通常表示为 32 位十六进制字符串。

  • 输出长度：128 位 / 32 个十六进制字符
  • 安全性：已被证明存在碰撞漏洞（2004 年王小云团队），不应用于安全场景
  • 现有用途：文件完整性校验、非安全场景的数据指纹、缓存键生成

SHA 安全散列算法

SHA（Secure Hash Algorithm）是由 NIST 发布的一系列哈希算法标准。

算法输出长度安全性状态
SHA-1160 位 / 40 字符已被攻破（2017 年 Google SHAttered）已弃用，不推荐
SHA-256256 位 / 64 字符目前安全广泛使用（TLS、区块链、数字证书）
SHA-384384 位 / 96 字符目前安全高安全需求场景
SHA-512512 位 / 128 字符目前安全高安全需求场景

AES 高级加密标准

AES（Advanced Encryption Standard）是目前最广泛使用的对称加密算法，由 NIST 于 2001 年发布，取代了 DES。AES 基于 Rijndael 算法，支持 128/192/256 位密钥长度。

AES 工作模式

模式全称特点是否需要 IV
ECBElectronic Codebook最简单，相同明文块产生相同密文块，不推荐否
CBCCipher Block Chaining每块与前一块密文异或后加密，最常用是
CTRCounter将块密码转为流密码，支持并行加密是（Nonce）
CFBCipher Feedback将块密码转为流密码，支持小于块大小的数据是

填充方式

  • PKCS7：填充 n 个字节，每个字节值为 n（最常用，RFC 5652）
  • ZeroPadding：用 0x00 填充至块大小，无法区分数据末尾的 0x00
  • NoPadding：不填充，要求输入数据长度必须是块大小（16 字节）的整数倍

安全建议

  • 密码存储应使用专用算法（bcrypt、scrypt、Argon2），而非 MD5/SHA
  • AES 加密推荐使用 CBC 或 GCM 模式，避免使用 ECB
  • IV 应为随机生成，每次加密使用不同的 IV
  • 密钥长度推荐 256 位以获得最高安全性

关于 Unix 时间戳

什么是 Unix 时间戳？

Unix 时间戳（Unix Timestamp），也称为 POSIX 时间或 Epoch 时间，是一种时间表示方式。它定义为从 UTC 1970年1月1日 00:00:00（称为 Unix 纪元，Unix Epoch）起至目标时刻所经过的秒数，不计闰秒。

基本特性

  • 起点：UTC 1970-01-01 00:00:00（Unix Epoch）
  • 单位：秒（部分系统使用毫秒、微秒或纳秒）
  • 时区无关：时间戳本身是 UTC 时间，与时区无关，同一时刻全球的 Unix 时间戳相同
  • 不计闰秒：Unix 时间不包含闰秒，每天固定为 86400 秒
  • 有符号整数：负值表示 1970 年之前的时间

精度与存储

精度单位示例值常见使用场景
秒级s1700000000大多数 Unix/Linux 系统、C 语言 time()
毫秒级ms1700000000000JavaScript Date.now()、Java System.currentTimeMillis()

2038 年问题

使用 32 位有符号整数存储的 Unix 时间戳，最大值为 231−1 = 2147483647，对应 UTC 2038年1月19日 03:14:07。超过此时刻后将发生整数溢出。现代 64 位系统已使用 64 位整数存储时间戳，可表示约 2920 亿年，不存在此问题。
